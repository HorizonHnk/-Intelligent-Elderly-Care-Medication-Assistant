🔌 UPLOAD ESP32 CODE (5 STEPS)
═══════════════════════════════════════════════════════════════════

1. Open Arduino IDE 2.x

2. File → Open:
   C:\Users\Dell\Documents\-Intelligent-Elderly-Care-Medication-Assistant\arduino\medicare_esp32_api\medicare_esp32_api.ino

3. Install libraries (Tools → Manage Libraries):
   - WebSockets by Markus Sattler
   - ArduinoJson by Benoit Blanchon (v6.x)
   - LiquidCrystal_I2C by Frank de Brabander

4. Update lines 20-24 with YOUR credentials:
   const char* ssid = "YOUR_WIFI_NAME";
   const char* password = "YOUR_WIFI_PASSWORD";
   const char* geminiApiKey = "YOUR_GEMINI_API_KEY";

5. Upload:
   - Tools → Board → ESP32 Dev Module
   - Tools → Partition Scheme → "Minimal SPIFFS (1.9MB APP)"
   - Tools → Port → [Your COM Port]
   - Click Upload (Ctrl+U)

⚠️ ABOUT THE MICROPHONE WARNING
═══════════════════════════════════════════════════════════════════

You'll see this warning:
"⚠️ Microphone requires HTTPS or localhost"

THIS IS NORMAL! It's because:
- Website is HTTPS (Netlify - secure)
- ESP32 is HTTP (local device - not secure)
- Browsers warn about "mixed content"

✅ SOLUTION (One-Time Setup):
When you first connect to ESP32:
1. Click the shield icon (🛡️) in browser address bar
2. Click "Load unsafe scripts" or "Allow"
3. Done! Everything works now.

📖 Full explanation: HTTPS_MICROPHONE_EXPLANATION.md

ALL FEATURES WORK AFTER THIS ONE-TIME CLICK! ✅
